1 - Servidor web y transferencia de archivos

¿Qué es un servidor web y cuáles son los elementos clave de su configuración para garantizar un acceso seguro?

Un servidor web es un software que recibe peticiones de clientes (navegadores) y entrega contenido. Para garantizar la seguridad, es fundamental configurar el uso de HTTPS (activando SSL/TLS), gestionar certificados digitales válidos, establecer autenticación de usuarios y definir reglas de control de acceso (por ejemplo, mediante hosts virtuales y módulos de seguridad).

¿Qué es la implantación de la arquitectura web?

La implantación de la arquitectura web es el proceso de organizar y desplegar los distintos recursos (servidores, bases de datos, CDN, funciones serverless, etc.) siguiendo un diseño estructurado que garantice:

  • Facilidad de uso y accesibilidad para el usuario final.
  • Utilidad y mantenibilidad del sistema, facilitando actualizaciones y correcciones.
  • Adaptabilidad a distintos entornos y requisitos cambiantes.
  • Escalabilidad, tanto horizontal (añadiendo más máquinas al conjunto) como vertical (añadiendo recursos—CPU, RAM—a cada máquina).
  • Portabilidad, de modo que la misma aplicación pueda moverse entre diferentes infraestructuras físicas o en la nube sin necesidad de reescribir código.

¿Qué modelos de arquitectura web y cuáles son sus características?

Los modelos de arquitectura incluyen el cliente/servidor y el modelo de tres capas.

Resumen simplificado de la arquitectura cliente/servidor

En su forma más básica, una página web opera bajo el modelo cliente/servidor:

  • Clientes: los navegadores (o programas) que usan los usuarios.
  • Servidor: una máquina dedicada que recibe las peticiones de los clientes y envía la información solicitada.

Cuando un usuario pide una página, el navegador envía una solicitud al servidor web. Éste responde entregando archivos, normalmente:

  • HTML estático (texto con marcas de hipertexto).
  • Archivos multimedia (imágenes, audio, vídeo).

El navegador interpreta ese contenido y lo muestra en pantalla. En este esquema estático, todos los usuarios reciben exactamente la misma página, que solo puede cambiar si los programadores modifican manualmente los archivos en el servidor.

Modelo de tres capas

El modelo de tres capas es una arquitectura común en aplicaciones web que guarda relación con el patrón Modelo-Vista-Controlador (MVC). Aunque no son equivalentes, ambos promueven la separación de responsabilidades dentro del sistema. Este modelo distingue tres elementos fundamentales:

  • Modelo: Representa los datos y su lógica. Vista y controlador actúan como puente entre los datos internos del sistema y la comprensión que tiene el usuario, permitiendo consultar y modificar información sin conocer cómo se almacena internamente.
  • Vista: Es la representación visual del modelo, adaptada para que el usuario interactúe con la información. Puede mostrar todo el modelo o solo una parte relevante.
  • Controlador: Es el componente que interpreta las acciones del usuario (clics, formularios, etc.) y responde a ellas, actualizando el modelo o la vista según corresponda.

Capas del modelo de tres capas

  • Capa 1. Navegador web: Forma parte de la capa de presentación. Permite al usuario visualizar datos y enviar formularios que serán procesados por el servidor de aplicaciones.
  • Capa 2. Servidor de aplicaciones web: Contiene la lógica de negocio. Recibe peticiones del cliente, consulta la base de datos, genera contenido dinámico y envía las páginas resultantes al navegador. El servidor web está normalmente integrado en esta capa.
  • Capa 3. Servidor de base de datos: Es la capa encargada del almacenamiento de datos. Responde a las consultas del servidor de aplicaciones y representa lo que en MVC se conoce como el modelo.

¿Qué son los criterios de funcionalidad en una implantación Web?

  • Disponibilidad: el servicio debe estar accesible siempre que el usuario lo necesite.
  • Rendimiento: respuestas rápidas, con tiempos de carga y procesamiento adecuados.
  • Escalabilidad: capacidad para crecer (más tráfico o datos) sin degradar el servicio.
  • Fiabilidad: operativa continua con mínimos errores o caídas.
  • Seguridad: protección contra accesos no autorizados y vulnerabilidades.
  • Mantenibilidad: facilidad para actualizar, corregir y mejorar la aplicación.
  • Usabilidad: interfaz clara y coherente que facilite la interacción del usuario.

Protocolos HTTP y HTTPS

HTTP (HyperText Transfer Protocol)

  • Función: Protocolo de comunicación para la transferencia de páginas web (HTML), imágenes y otros recursos entre cliente (navegador) y servidor.
  • Puerto por defecto: 80
  • Características principales:
    • Texto plano: las peticiones y respuestas viajan sin cifrado.
    • Basado en modelo petición-respuesta: el cliente envía un método (GET, POST, etc.) y el servidor responde con un código de estado (200, 404, etc.) y un cuerpo (HTML, JSON, etc.).
    • Stateless: cada petición es independiente; el servidor no recuerda estados anteriores.

HTTPS (HTTP Secure o HTTP sobre TLS/SSL)

  • Función: Misma lógica que HTTP, pero con una capa de seguridad que cifra toda la comunicación.
  • Puerto por defecto: 443
  • Características principales:
    • Cifrado TLS/SSL: asegura confidencialidad (nadie puede leer los datos), integridad (no se modifican en tránsito) y autenticidad (el servidor es quien dice ser).
    • Handshake: antes de intercambiar datos HTTP, cliente y servidor negocian algoritmos de cifrado y validan certificados digitales.
    • Requisitos: certificado SSL/TLS emitido por una Autoridad de Certificación (CA) o Let’s Encrypt para entornos gratuitos.

Certificados SSL/TLS

SSL vs. TLS

Son ficheros digitales que autentican la identidad de un servidor y cifran la comunicación para proteger los datos en tránsito.

  • Origen: SSL es el protocolo inicial (ahora obsoleto); TLS es su evolución segura.
  • Seguridad: TLS corrige vulnerabilidades de SSL y usa cifrados modernos.
  • Rendimiento: TLS (especialmente 1.3) tiene un “handshake” más rápido y eficiente.
  • Uso actual: SSL ya no se usa; siempre se implementa TLS (mínimo 1.2, ideal 1.3).

¿Qué es un Certificado digital?

“Fichero informático firmado electrónicamente por un prestador de servicios de certificación.”

Tiene una estructura de datos que contiene información sobre la entidad. La firma de la estructura de datos agrupa la información que contiene de forma que no puede ser modificada sin que esta modificación sea detectada.

¿Qué es un Certificado de clave pública?

Asocian una clave pública a alguna representación de la identidad de la entidad. La clave privada se asocia a la clave pública y asume que está en posesión de dicha entidad.

Se puede incluir otra información relevante como periodo de validez, identificador de los algoritmos para los cuales se puede usar la clave pública, restricciones, etc.

Para asegurar la confidencialidad del mensaje se utilizan técnicas y sistemas criptográficos. El cifrado consiste en aplicar un algoritmo para cifrar un mensaje reconocible y que este solo pueda ser descifrado con este mismo algoritmo y su clave.

Hay dos opciones básicas en el cifrado digital:

  • Clave simétrica: Se utiliza una sola clave para cifrar y descifrar el mensaje. Proceso más rápido, pero más expuesto al tener que enviar la clave.
  • Clave asimétrica: Cada usuario tiene una pareja de claves, una privada y otra pública. Proceso más lento, pero más seguro al no enviar la clave.

¿Qué es un Certificado SSL?

“Es un certificado digital que autentica la identidad de un sitio web y habilita una conexión cifrada. La sigla SSL significa Secure Sockets Layer (capa de sockets seguros), un protocolo de seguridad que crea un enlace cifrado en un servidor web y un navegador web.”

Puertos de comunicación y firewall

Puertos de comunicación

Los puertos son identificadores numéricos (0–65 535) que permiten a un mismo equipo ofrecer múltiples servicios de red simultáneamente. Cada servicio “escucha” en su puerto asignado para recibir conexiones:

  • Bien conocidos (0–1023): HTTP (80), HTTPS (443), SSH (22), FTP (21)…
  • Registrados (1024–49 151): servicios de aplicaciones específicas.
  • Dinámicos/privados (49 152–65 535): asignados de forma temporal por el sistema operativo.

Firewall

Un firewall es un filtro entre dos redes (por ejemplo, tu servidor e Internet) que decide qué tráfico permitir o bloquear según reglas predefinidas:

  • Filtrado de paquetes (stateless): examina cada paquete de forma individual (puerto, IP, protocolo).
  • Inspección con estado (stateful): mantiene contexto de conexiones activas, ofreciendo mayor precisión.
  • Firewall de aplicación (WAF): analiza tráfico a nivel HTTP/S para proteger contra ataques web (inyección SQL, XSS, etc.).

Parámetros de configuración: permisos, usuarios, cuotas, tiempo de espera

Permisos

  • Definen qué operaciones puede realizar cada entidad (archivo, proceso, usuario).
  • Se gestionan a nivel de sistema de ficheros (lectura, escritura, ejecución) y a nivel de aplicación (roles y privilegios).
  • Buena práctica: aplicar el principio de mínimo privilegio: dar solo los permisos estrictamente necesarios.

Cuotas

  • Límites asignados a usuarios o aplicaciones sobre recursos (espacio en disco, número de procesos, conexiones simultáneas, ancho de banda).
  • Evitan que un solo usuario agote recursos críticos y aseguran calidad de servicio al resto.
  • Ejemplo: máximo 5 GB de almacenamiento por cuenta, 100 conexiones concurrentes por IP.

Tiempo de espera (timeouts)

  • Idle timeout: tiempo de inactividad tras el cual se cierra la sesión.
  • Connection timeout: límite para establecer la conexión.
  • Request timeout: tiempo máximo para procesar una petición.

Esto se aplica al apartado de FTP (tener en cuenta).

Tipos de usuarios y accesos al servicio

Existen tres formas de acceder a un servicio de transferencia de archivos, cada una de las cuales define un tipo de usuario:

  • Acceso anónimo: acceso libre sin restricciones. No se solicita autenticación. Solo permite descargar archivos.
  • Acceso de Invitado: requiere un usuario genérico con acceso restringido. Es sencillo de administrar porque ya está predefinido en el sistema.
  • Acceso con usuario: requiere autenticación con nombre y contraseña. Requiere mantenimiento y asignación de permisos. Es el sistema utilizado en servidores compartidos de páginas web para evitar modificaciones no autorizadas.
